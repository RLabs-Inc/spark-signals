§Spark Signals

A fine-grained reactive signals library for Rust.

This library provides reactive primitives similar to SolidJS, Svelte 5, or Preact Signals, but designed for Rust applications like TUI frameworks, game engines, and simulations.

§Quick Start

use spark_signals::{signal, derived, effect};

// Create a signal (reactive value)
let count = signal(0);

// Create a derived value (computed from signals)
let count_clone = count.clone();
let doubled = derived(move || count_clone.get() * 2);

// Create an effect (runs when dependencies change)
let doubled_clone = doubled.clone();
let _effect = effect(move || {
    println!("Doubled: {}", doubled_clone.get());
});

// Changing the signal triggers the effect
count.set(5); // Prints: "Doubled: 10"

§Core Concepts

  • Signal: A reactive container for a value. Reading inside a reaction creates a subscription.
  • Derived: A computed value that recomputes when its dependencies change.
  • Effect: A side effect that re-runs when its dependencies change.

§Memory Management

Effects are automatically cleaned up when dropped. You don’t need to manually dispose them.

use spark_signals::{signal, effect};

let count = signal(0);

{
    let count_clone = count.clone();
    let _effect = effect(move || {
        println!("Count: {}", count_clone.get());
    });
    count.set(1); // Effect runs
} // _effect dropped here - automatic cleanup

count.set(2); // Effect does NOT run - it's been cleaned up
